# 题小助上线前最终验收计划

目标：按正式商业上线标准，对已有角色、功能、页面、按钮和关键数据链路做最终验收。支付本期仍按延期上线处理，只验证入口隐藏、现有学习/题库能力不受影响。

## 验收角色

- 学生端：手机、平板、电脑三端视口都必须可用，重点验证首次进入、注册登录、首页、练习、题库商城、拍题、错题、我的档案。
- 运营后台：电脑端使用，重点验证管理员登录、仪表盘、用户档案、习题内容、AI 出题、题库知识点、学习记录、收费积分、系统权限。
- 平台服务：验证生产 ready、支付延期策略、真实 AI 配置、题库数据、后台授权导出等基础链路。

## 自动验收命令

```bash
npm run audit:launch-ui
```

这条命令会先构建学生端和后台，再启动隔离的生产模式后端，用 Playwright 执行浏览器验收。测试数据写入临时文件，不污染本地正式数据。

## 当前覆盖

- 学生端三端适配：手机 `390x844`、平板 `768x1024`、电脑 `1280x900`。
- 学生端核心流程：年级选择、昵称设置、正式账号注册、首页、练习中心、完成一轮练习、拍题手动输入、提交批改、错题页、学习档案、学习记录弹窗、收藏弹窗。
- 后台核心流程：管理员登录、七个后台菜单逐页进入、页面标题检查、搜索/退出等全局按钮存在性、AI 出题按钮真实生成预览。
- 服务补充验收：`/api/health`、隔离环境 `/api/ready` 返回状态记录、支付延期配置、知识点讲义 PDF 授权导出接口。

## 仍需人工复核

- 真实移动设备手感：触摸区域、软键盘遮挡、刘海屏安全区、长列表滚动是否顺手。
- 真实 OCR：需要生产 OCR 服务后，用真实拍照图片验证识别质量。
- 真实部署环境：云 PostgreSQL、域名、HTTPS、对象存储、监控告警接入后，还要做一次线上 smoke。
- 支付：本期不碰支付，上线前只确认支付入口隐藏且学习功能不依赖支付。
